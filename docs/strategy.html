BTC Adaptive Trading System

テクニカル + ファンダメンタルの多層制御で、あらゆる相場環境に適応する自動売買システム

このシステムの考え方
相場には「上昇」「横ばい」「下落」の3つの局面がある。それぞれに得意な戦略は違う。

上昇局面: トレンドに乗って持ち続ける戦略（ema_don）が主役
横ばい局面: 安い時に買って高い時に売る逆張り戦略（bb）が主役
下落局面: 何もしない。現金で待機

この切り替えを50日移動平均線（SMA50）で自動判定し、さらにゴールド・米国債・市場心理の3つの指標で判定精度を高めている。

Dynamic Weight Control

過去50日の平均価格（SMA50）と現在価格の乖離率で「今はどんな相場か」を判定し、各戦略への資金配分を毎日自動で調整する。上昇中は順張りに多く配分し、下落中は全額退避する。

Fundamental Score Adjustment

3つのマクロ指標から算出したファンダスコアで、レジーム判定のタイミングを微調整する。
数字は「BTC 30日後リターンとの相関」。+1.0なら完全に連動、-1.0なら完全に逆。0.3前後は「弱いが統計的に有意」で、公開データから得られる中ではかなり良い水準。

仕組み
Early Transition（早期転換）: テクニカルがレジーム境界付近にいる時、ファンダで数日早く判定を切り替える。
例: SMA50乖離+3% + ファンダ弱気 → uptrendではなくrangeとして扱う

Boost（確信度ブースト）: テクニカルもファンダも強気なら、ema_don比率を70% → 90%にブースト。
例: uptrend + ファンダスコア > 0.5 → ema_don 90% / bb 10%

Daily Decision Flow

毎日1回、以下のステップで売買判断を行う。

1
データ取得
BTC日足（Binance）+ ゴールド・米10年債・Fear&Greed を取得
2
相場判定
SMA50との乖離率で 上昇 / レンジ / 下落 を判定。ファンダスコアで境界付近の判定を微調整。
3
資金配分
レジームに応じて3戦略に資金を振り分ける。下落判定なら全額を現金退避。
4
シグナル生成
各戦略がテクニカル指標（BB, EMA, ドンチャン）から売買シグナルを算出。
5
注文実行
買い/売りシグナルが出たら指値注文。約定確認後にストップロスを設定。未約定ならキャンセル。
