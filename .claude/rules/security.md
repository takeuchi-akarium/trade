# セキュリティルール

## 絶対に行わないこと

- APIキー・シークレット・パスワードをコードにハードコード
- `.env` ファイルをコミット
- ユーザー入力を検証なしにSQL・シェルコマンド・HTMLに渡す（SQLインジェクション・XSS・コマンドインジェクション防止）
- `eval()` の使用

## 環境変数

- 機密情報は必ず環境変数経由で参照: `process.env.API_KEY`
- `.env.example` に必要な変数名（値なし）を記載する

## 依存関係

- 依存関係を追加する際は既知の脆弱性がないか確認する
- `npm audit` で定期的にチェック
